Exceptions Demo V2

Details

  • Demo running with profile: profiles
    To change the profile, edit Main.java and change the value of the ActiveProfiles data-member.
  • See Blog for more details.
  • Profile discussion in the Blog.
  • Demo uses the Thymeleaf view generation system to generate HTML pages (a nice alternative to JSP that cn be configured by Spring Boot automatically).

Spring-Boot sets up error view as the default error-page and we are using Thymeleaf to define it via the error.html template (if you don't configure the Spring Boot error page you see the so-called "Whitelabel Error Page").

Without Spring Boot you would see your containers default error page. Rename error.html to error2.html to see the difference.

The previous version of this demo tried to show everything on one page. It has been broken down into different pages for clarity.

The Demo

  • Local Exception Handling - Exceptions handled in the Controller.
    URLs on this page are handled by the demo1.web.ExceptionHandlingController which handles its own exceptions using its own @ExceptionHandler methods.
  • Global Exception Handling - Exceptions handled by a Controller Advice.
    URLs on this page are handled by the demo2.web.ControllerWithoutExceptionHandlers which relies on a Controller Advicedemo2.web.GlobalExceptionHandlingControllerAdvice to handle any exceptions thrown.
  • Simple Mapping Exception Resolver - Exceptions handled by exception resolving bean
    Exceptions raised by ExceptionThrowingController
  • No Handler for Exceptions
    For comparison, the same exceptions as previous option, but with the SimpleMappingExceptionResolver disabled to see what you get by default. Exceptions are not totally unhandled because Server 500 errors are caught by Spring Boot's default error page - mapped to error.html.
